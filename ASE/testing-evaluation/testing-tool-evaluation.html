Meridian ASE Testing Evaluation

Meridian is a microservice-based logistics platform with an API gateway and separate delivery, route, driver, vehicle, assignment, tracking, and user services. For the ASE testing task, the team selected xUnit for backend unit testing and k6 for API-gateway load testing.

Course: Advanced Software Engineering Group Size: 4 Tools: xUnit + k6 Demo Style: Local-first

Why These Tools

  • xUnit matches the existing Meridian backend stack, so the team can extend real project tests instead of creating a disconnected demo setup.
  • k6 already exists in the backend repository for gateway traffic, making it a natural fit for performance-focused ASE work.
  • The two tools come from different categories: unit testing and load testing.

Current Readiness Snapshot

  • The backend already contains 7 xUnit test projects and about 220 discovered tests.
  • RouteService.Tests and TrackingService.Tests were already locally runnable.
  • The ASE work hardened UserService.Tests/AuthValidationTests.cs so it no longer depends on ConnectionStrings:UserDb for the auth demo path.
  • The backend already had load-tests/dispatcher-session.js and load-tests/report-generation.js; ASE work adds local smoke profiles, env templates, wrappers, and evidence flow.
  • Local machine requirement: k6 must be installed before the load demos can run.

ASE Scope In The Repository

The tracked ASE hub is Meridian-Backend/ASE/testing-evaluation/. ASE-only deliverables live there, while real test code remains in the existing test projects and load-tests/.

  • ASE documentation: README, team work, demo plan, runbook, evidence guidance, and this HTML file
  • Unit-test implementation: existing xUnit projects under src/*/*.Tests
  • Load-test implementation: existing scripts under load-tests/
